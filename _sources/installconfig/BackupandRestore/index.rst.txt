=======================================
Back Up and Restore
=======================================

.. include:: /shared/ProductLabels/CustomerManaged_Label.rst

This section contains information about backing up and restoring the Alation application on customer-managed (on-prem) instances.

Alation either automatically creates backups daily (default) or runs the backup process on a user-defined custom schedule.

The backup and restoration steps require access to the backend of the Alation server.

This section covers:

- :doc:`Information about the backup process and tools </installconfig/BackupandRestore/BackupV1AndV2/index>`
- :doc:`The backup steps </installconfig/BackupandRestore/CreateBackupsManually>`
- :doc:`The restoration steps </installconfig/BackupandRestore/RestoreFromBackup/index>`
- :doc:`Information on configuring and maintaining backups </installconfig/BackupandRestore/ConfigureBackups/index>`
- :doc:`Recommendations on how to restore the Alation Analytics </installconfig/BackupandRestore/RestoreFromBackup/RestoreAlationAnalytics>`
- :doc:`Recommendations on how to restore OCF components </installconfig/BackupandRestore/RestoreFromBackup/RestoreOCFComponents>`
