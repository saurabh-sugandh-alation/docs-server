Version 2022.2
===================

Refer to the tables below for information on supported database, file system, and BI tool types and versions.

.. important::

    Alation has certified support for versions in this matrix with release 2022.2. If your Alation instance uses a different release, please refer to the Support Matrix corresponding to this release.

Definitions
-----------

**Compose**: the Alation query tool for authoring, running, and publishing SQL queries to the data catalog.

**Lineage**: visual representation of how tables, views, and BI reports are derived from source tables.

**Metadata Extraction**: extraction of metadata from a source into the catalog with the goal of synchronizing them and providing catalog users with a representation of the data contained in this source. Metadata extraction can be run manually or on an automatic schedule that depends on your organization's needs and usage.

**Popularity**: an indicator of the popularity (intensity of use) of a data object, such as a table or a column. Popularity is calculated through an algorithm that analyzes both the results of Query Log Ingestion and Compose usage to determine the number of Alation users that have used the object and the number of times the object has been used.

**Query Log Ingestion (QLI)**: processing of a selection of database query logs by Alation in order to calculate Popularity, Top Users, and supply JOIN information to the catalog.

**Sampling**: creation of data samples for the cataloged data sources. When users perform sampling, Alation scans the first 10,000 rows of a table. A data sample is a random sample of 100 rows out of those 10,000 scanned rows intended to provide an insight into data in tables.

**Profiling V2** or *deep column profiling*: retrieval of a data sample based on a default or custom query and statistical calculations on this sample. The profiling results are represented on the Frequency Distribution chart for a column.
